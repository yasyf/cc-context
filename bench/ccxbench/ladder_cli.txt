## Compact Context (ccx)

`ccx` is the DEFAULT for reading code, finding symbols, searching, and reviewing
diffs. It returns token-bounded output (signatures + line numbers, explicit overflow,
never silent truncation) instead of raw dumps. Run every op as a `ccx ...` shell
command through Bash. Reach for ccx before Read, cat, sed, git diff, ls -R, or find —
the guard hooks block those on anything token-heavy.

1. Orient a repo → `ccx repo overview`
2. "How does X work / where is Y" (intent) → `ccx code search "<question>"`
3. A specific symbol (def + callers + callees) → `ccx code symbol <name>` (alias `ccx code grok`)
4. Literal / structural text → `ccx code grep <text> [--glob G]`
5. List files → `ccx repo find "<glob>"`
6. Read a file → `ccx code outline <file>` first, then `ccx code read <file> --section A-B` for the
   part you need (whole file: `ccx code read <file> --full`)
7. Review changes → `ccx vcs diff [src]` (structural, jj-aware; exact hunks: `git diff -- <file>`)
