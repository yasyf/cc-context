## Reading code efficiently

Favor the fewest tokens that still carry the answer. Navigate before you dump, and
never read a whole file when a scoped slice carries what you need. Scope every step
to the smallest surface that answers the question and stop once you have it.

For a targeted or enumeration question, search by content first. Chain the commands
you need in ONE shell invocation, and pipe output through `grep`/`sort`/`uniq`/`awk`
so only what you need reaches you — never surface intermediate noise.

1. Find text/defs across a tree → `grep -rn <text> <dir>` scoped to the paths that
   matter (one scoped search for the whole set; do NOT read every file).
2. "How does X work / where is Y" (intent) → grep by content, then open only the
   lines the search points at.
3. For a specific symbol, grep its definition and every line that calls it, then
   read only those lines.
4. Enumerate a set spanning many files → one scoped `grep`, or a short
   `python3 -c`/`awk` pipeline that filters and prints just the members.
5. Read a file → a bounded read (offset/limit) of just the part you need;
   read the whole file only when it is genuinely small.
